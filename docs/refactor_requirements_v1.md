# 리팩토링 요구사항 정리본 v1 (확정)

## 1. 범위
- 1차 범위: 엔드투엔드(백엔드 파이프라인 + 프론트엔드 UI)
- MVP 우선순위: 특허 초안 1~7 모듈 전부 포함
- 일정: 1주 내 완료 목표
- 전환 방식: 저녁 시간대 점검 배포(무중단 필수 아님)

## 2. 핵심 설계 원칙
- 분류 방식: 규칙 기반 + LLM 보조(설명 생성)
- 모델 전략: OpenRouter 고정
- 문서화: 템플릿 기반 정식 문서화 포함
- 출력 품질: "AI 티"가 덜 나도록 사용자 양식 중심 출력

## 3. 데이터/지식베이스
- 정책/인증/RFP 기준: 초기에는 일반 기준을 바탕으로 수집/정규화하고 LLM 보조 추론 적용
- 갱신 정책: 초기 지식베이스 + 사업계획 생성/상담 결과 반영 시 갱신
- 운영 주체: 시스템 + 상담/운영자 피드백 반영

## 4. 입출력
- 입력: 텍스트 + 파일/폴더 업로드
- 지원 포맷: `pdf`, `docx`, `hwp`, `hwpx`, `ppt`, `pptx`, `txt`, `excel`
- 출력: 브라우저 HTML 뷰 + Markdown + PDF 다운로드

## 5. 품질/검증
- KPI: 전문가 판정 일치율, 추천 수용률, 누락률
- DoD: 테스트 통과 + 샘플 케이스 재현 + 성능 기준 충족
- 테스트 범위: 단위/통합/회귀/부하

## 6. 운영/정책
- 개인정보/민감정보: 이번 스프린트에서는 별도 정책 적용 제외
- 멀티테넌시/권한 강화: 이번 스프린트 신규 강화 요구 없음
- 마이그레이션: 기존 데이터 마이그레이션 요구 없음

## 7. 비고
- 일부 도메인 임계치/점수 산식/정책 기준은 인터넷 조사 기반으로 정의 필요
- 본 문서는 `docs/refactor_additional_questions.md`의 답변을 실행 가능 형태로 재정리한 문서임
